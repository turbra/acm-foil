ACM Foil is inspired by [Greg Procunier's (`gprocunier`)](https://github.com/gprocunier) [Blastwall project](https://github.com/gprocunier/blastwall.git). Blastwall demonstrates hardened fleet management for RHEL with IdM, with a path for bringing that same hardened disposition to OpenShift workloads. ACM Foil extends that model into ACM-driven OpenShift fleet management.

## Quick Start

<Tabs>
  <TabItem value="deploy" label="Deploy" default>

```bash
oc apply -f apps/hub/argocd-application.yaml
```

  </TabItem>
  <TabItem value="target" label="Target a cluster">

```bash

```

  </TabItem>
  <TabItem value="validate" label="Validate">

```bash
oc get applications.argoproj.io -n openshift-gitops spo-acm-policies-test
oc get policy,policyset,placement,placementbinding -n acm-spo-policies
```

  </TabItem>
</Tabs>

## Current Policy Flow

```text
GitHub repository
  -> OpenShift GitOps on the ACM hub
  -> ACM policy namespace
  -> PolicySet and Placement
  -> managed clusters labeled spo=true
  -> security policy resources
```

## What It Deploys

| PolicySet | Purpose |
| --- | --- |
| `policyset-blastwall-test` | Deploys Blastwall Security Profiles Operator resources and validation objects. |
| `policyset-spo-test` | Deploys a Red Hat CVE mitigation policy and a harmless SPO smoke profile. |

ACM Foil does not install the Security Profiles Operator. Target clusters must already have SPO installed when SPO-specific policies are enabled.
