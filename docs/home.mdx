ACM Foil is inspired by [Greg Procunier's (`gprocunier`)](https://github.com/gprocunier) [Blastwall project](https://gprocunier.github.io/blastwall/). Blastwall demonstrates hardened fleet management for RHEL with IdM, with a path for bringing that same hardened disposition to OpenShift workloads. ACM Foil extends that model into ACM-driven OpenShift fleet management.

## Current Policy Flow

```text
GitHub repository
  -> OpenShift GitOps on the ACM hub
  -> ACM policy namespace
  -> PolicySet and Placement
  -> managed clusters labeled spo=true
  -> security policy resources
```

## What It Deploys

| PolicySet | Purpose |
| --- | --- |
| `policyset-blastwall-test` | Checks SPO `RawSelinuxProfile` readiness, then deploys Blastwall Security Profiles Operator resources and validation objects. |
| `policyset-spo-test` | Installs the Security Profiles Operator, deploys a Red Hat CVE mitigation policy, and creates a harmless SPO smoke profile. |

ACM Foil installs the Security Profiles Operator on selected managed clusters through ACM `OperatorPolicy`. Clusters must have OLM and access to the Red Hat operator catalog.
